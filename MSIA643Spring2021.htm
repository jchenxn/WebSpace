1. Description: This course is about database security and auditing. It consists of three blocks: database security fundamentals, article readings, secuirty and auditing implementation in Oracle. Students will be exposed to both discretionary and mandatory security models. The course will focus on user administration, discretionary access control, virtual private database, and various auditing models. Other topics covered include SQL injection, statistical database security, data encryption. Students will learn how to implement database security on modern business databases using practical scenarios and step-by-step examples.

2. Objectives: Upon successfully completing this course, the students will have a solid understanding of database security and auditing models and techniques. Be able to implement database application security and auditing models.

3. Learning Outcomes: Be able to describe the security challenges and threats in database systems; Be able to demonstrate knowledge of basic information security architecture, database security models, and database auditing principles and methods; Be able to apply database access control methods; Be able to implement database security in business database applications; Be able to configure and conduct database auditing.

4. Required Textbook and Articles: Sam Afyouni, Database Security and Auditing: Protecting Data Integrity and Accessibility. Course Technology Incorporated, 2006, ISBN 0-619-21559-3. The articles are on the course D2L website.

Individual and Group Projects: You will complete four to six projects, of which some are individual projects. No late project/program is accepted unless you can provide official document of the extreme situation that prevented you from completing your project on time. No excuse is accepted after the project deadline.

Article Reviews (AR): You will read six assigned articles. Out of the six, you will choose three of them to write a 2 to 3 page long summary and review per each article.
Your Article Review should include a summary of the article's core contents, arguments, conclusions AND your critical review after your own research on the topic. For example, new developments (technologies, theories) on the topic can be included in your critical review.

Quizzes: Quizzes are of code writing or short answer type. They cover assigned materials and reading articless.
Exams: The exam questions are of short answer type and problem solving, which cover the lecture materials, projects, and, articles.
