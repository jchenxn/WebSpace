TUTORIAL 1 INTRODUCTION TO ORACLE 12c

IS 443/543 Database Design & Implementation

 

Oracle is one of the most widely used database management system in the world. It is an Object-Relational DBMS and runs on virtually every kind of computer, from laptops, PCs, to minicomputers and giant mainframes. It is designed to meet the requirements of the network computing and to provide reliable and secure data management for applications ranging from small departmental applications to high-volume on-line transaction systems or query-intensive data warehouse applications.

 

Oracle Database 12c, released in 2013, was designed for the Cloud, featuring a new Multitenant architecture, In-Memory column store, and support for JSON documents. Oracle 12c allows an enterprise to have one super database (=container; CDB) and one or many sub databases (= pluggable DBs; PDB).

SQL Data Definition & Data Manipulation

SQL Plus is a very powerful language that performs various tasks ranging from defining database structure to ad hoc queries. We will use an exercise to demonstrate how to create a Oracle database structure, populate a table, and alter table structure. The situation we wish to represent here is that of students enrolled in courses. For each student we need to store student number, name, street address, city, and enrolment date. A course code, course name, and the teacher's name describe each course.

Relational Schema

COURSE(Cou_code,Cou_Name,Cou_Teacher)

STUDENT(Stu_No,Stu_LName, Stu_Fname, Stu_Street, Stu_City, Stu_Enroll_Date, Stu_GPA)

ENROLLMENT(Cou_Code, Stu_No, Grade)

2. Modifying a table definition i.e. adding or deleting a column or a relationship is easy using Oracle SQL.

· Use ALTER TABLE .... ADD to add a new field to the table. You specify the field name, data type, and size. For example, the following statement adds a 25-character Text field called Notes to the Employees table:

ALTER TABLE Employees ADD (Notes Varchar2(25));

· Use DROP CONSTRAINT to delete a foreign key constraint. You specify the constraint name following the CONSTRAINT reserved word.

ALTER TABLE Enroll

DROP CONSTRAINT Enroll_CouCode_fk;

 

· Use ADD CONSTRAINT to add a foreign key constraint.

ALTER TABLE Enroll

ADD CONSTRAINT Enroll_CouCode_fk FOREIGN KEY(Cou_Code) REFERENCES

Course(Cou_Code);

· Use ALTER TABLE ... MODIFY to change data type and size of a column.

ALTER TABLE Student

MODIFY (Stu_Lname VarChar(30));

 

3. Use the Insert command to add data to the defined tables.

INSERT INTO COURSE

            VALUES ('IS365', 'Computer Systems', 'Giles');

- Check to see if it was inserted:

SELECT * FROM Course;

This insertion is not committed. It can be rolled back. Try the following:

Rollback;

SELECT * FROM Course;

To make the insertion permanent, type

COMMIT;

 

4. Once data are in the tables they may be altered using the UPDATE statement. For example to change the teacher for IS443 from JChen to Randy

            UPDATE Course

            SET Cou_Prof = 'Randy'

            WHERE Cou_Code = 'IS443' ;

i) change the course name of IS 250 to "Introduction to Computer Programming"

ii) change the city of students who live in St Cloud to Winona.

 

5. Data may also be deleted from tables when it is no longer needed. This may be done one row at a time or for an entire table. For example to delete student Smith from the STUDENT table

            DELETE FROM Student WHERE Stu_Lname = 'Smith' ;

Note i) All Smith's would be deleted.

         ii) This may cause referential integrity problems if Smith was actually enrolled in any subject.

 

6. Finally, a table may be deleted from a database.

The following example deletes the Student table from the database:

DROP TABLE Student;

The next example deletes the index MyIndex from the Employees table:

DROP INDEX MyIndex;

Note: An index file can be created using: CREATE INDEX Myindex ON TABLE Student(Stu_Lname);

 

7. Data Control Language

To grant Select privilege on table Student to user JSmith,

            GRANT SELECT ON Student to JSmith;

To revoke SELECT privilege on table Student from JSmith,

            REVOKE SELECT ON Student FROM JSmith;

 

To grant update privilege on table Student to everyone,

            GRANT UPDATE ON Student to PUBLIC;

 

To grant all privileges on Course table to Ted and Nancy;

            GRANT ALL PRIVILEGES ON Course to Ted, Nancy;

 

To find out to whom I have granted privileges,

            SELECT * FROM USER_TAB_PRIVS;
